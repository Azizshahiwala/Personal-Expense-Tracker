Client side python implementation:

import asyncio 
import websockets 

async def send_data():
    
    uri = "ws://localhost:8765"
    while True:
        async with websockets.connect(uri=uri) as websocket:

            #Since websocket is open, write input and send the data.
            data_in = input("Write something:")
            print("(Client)Data sent from client: ",data_in)
        
            response = await websocket.send(data_in)
            print("(Client)Response from server: ",response)


#Here, send_data is an entry point.
if __name__ == "__main__":
    asyncio.run(send_data())

What if its cross platform ?

Let react be client. It establishes connection to websocket upon load. and listens/sends messages to backend.
Next, from python backend, it subscribes to redis. (async pub/sub) and to websocket to close..

[React Frontend] ← WebSocket → [FastAPI Server] ← Async Pub/Sub → [Redis Broker]


Bridge 1: React to Python (The WebSocket Connection)
Your frontend establishes a persistent, two-way connection to the backend using the browser's native WebSocket API.

The React Code:

JavaScript

Key Note: This connection is strictly between one user's browser and one specific FastAPI server worker.

Bridge 2: Python to Redis (The Pub/Sub Connection)
FastAPI acts as the middleman. When it receives a WebSocket message from React, it uses an asynchronous Redis client to publish that message to a specific "channel" (the room).

The FastAPI Code:

Key Note: Redis is incredibly fast because it stores everything in RAM. It does not save the chat history permanently; it only handles the live routing of the messages.

The Complete Lifecycle (How a message travels)
When User A sends a message to User B, here is the exact sequence of events:

React (User A): Executes socket.send("Hello").
FastAPI (Worker 1): Receives "Hello". It optionally saves this to PostgreSQL so the history is permanent.
FastAPI (Worker 1): Executes redis_client.publish("room_V7YOME4584", "Hello").
Redis: Receives the publish command. It looks at its subscription list and instantly blasts "Hello" to all FastAPI workers listening to room_V7YOME4584.
FastAPI (Worker 2): Hears the broadcast from Redis. It looks at its active WebSockets and sees User B is connected to this room.
FastAPI (Worker 2): Pushes the message down the WebSocket to User B.
React (User B): The socket.onmessage event fires, updating the UI.
